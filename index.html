    2. Preenchendo regiões:
      • 2.1 Uma imagem que pode gerar esse problema é um "tabuleiro de xadrez", que entãi teria mais de 255 objetos desconexos (4 conectado). Para resolver esse problema podemos usar imagem colorida e então poderiamos ter 16777216 (256^3) rotulações diferentes, que é mais do que essa imagem pode produzir (ela pode produzir na ordem de Width*Height/2 objetos). Ou apenas setar a cor de todas elas para qual quer número diferente de 255 (e não setar para nobjetos).
